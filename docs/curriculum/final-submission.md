# 최종 결과물 제출 안내

> 오늘 실습한 결과물을 조직 GitHub 저장소에 올리기

## 왜 제출이 필요한가요?

오늘 배운 내용을 여러분의 실제 프로젝트에도 적용해, 조직 계정(`hyu-staff`) 아래 저장소로 결과물을 올려주세요. [소개](/)에서 이야기한 것처럼, 조직 계정에 모아두어야 담당자가 바뀌어도 프로그램이 남고, 다른 팀도 보고 발전시킬 수 있습니다.

## AI CLI에게 아래 프롬프트를 그대로 전달하세요

VS Code 오른쪽에 열리는 프롬프트(채팅) 창에 그대로 붙여넣는 방법입니다.

:::note[VS Code가 없다면]
[code.visualstudio.com](https://code.visualstudio.com/)에서 먼저 설치해주세요.
:::

1. VS Code를 열고, 상단 메뉴 **파일(File) → 폴더 열기(Open Folder...)** 로 결과물 폴더를 엽니다.
2. 오늘 설치한 AI CLI 도구의 VS Code 확장(예: Claude Code)을 실행하면, 화면 오른쪽에 프롬프트를 입력하는 채팅창이 열립니다.
   - 확장이 안 보이면 왼쪽 사이드바의 확장(Extensions) 아이콘에서 검색해 설치한 뒤, 아이콘을 눌러 오른쪽 창을 열어주세요.
3. 오른쪽 프롬프트 창에서, 아래 프롬프트의 **`[내 정보]`** 부분만 본인에 맞게 수정한 뒤 그대로 붙여넣고 Enter를 누릅니다.

:::tip[붙여넣기 전 확인]
프롬프트 안의 4가지 정보(성명 이니셜, 프로젝트명, 결과물 폴더 경로, 조직 계정)를 반드시 실제 값으로 바꿔주세요.
:::

```text
[내 정보] ← 본인에 맞게 수정하세요

성명 이니셜: hgd (예: 홍길동)
프로젝트명(예시): mail-automation
결과물 폴더 경로: C:\Users\내이름\Documents\내프로젝트
조직 계정: hyu-staff

[해야 할 일]

1. 내 PC에 git이 설치되어 있는지 확인하고, 없으면 winget으로 설치해줘.
2. git 사용자 이름(user.name)과 이메일(user.email)이 설정되어 있는지 확인하고, 없으면 나에게 물어봐서 설정해줘.
3. GitHub CLI(gh)가 설치되어 있는지 확인하고, 없으면 winget으로 설치해줘. 그다음 gh auth login으로 GitHub에 로그인하도록 안내해줘. (브라우저 인증 방식으로)
4. 조직 계정에 '이니셜_프로젝트명' 형식의 새 저장소를 만들어줘. (예: hgd_mail-automation, private으로)
5. 내 결과물 폴더를 git 저장소로 초기화하고, README.md가 없으면 프로젝트 소개(무엇을 하는 프로젝트인지, 어떻게 실행하는지)를 담은 README.md를 만들어줘.
6. 업로드하면 안 되는 파일(비밀번호, API 키, 개인정보가 담긴 파일)이 있는지 먼저 검사해서 알려주고, 있으면 .gitignore에 추가해줘.
7. 모든 파일을 커밋하고 조직 저장소에 푸시해줘.
8. 완료되면 저장소 주소(URL)를 알려주고, 브라우저에서 결과를 확인하는 방법을 설명해줘.

[주의사항]

명령을 실행하기 전에 어떤 명령인지 먼저 보여줘.
내 파일을 삭제하거나 덮어쓰는 작업은 하지 마.
잘 모르겠으면 진행을 멈추고 나에게 물어봐줘.
```

## 진행 중 이렇게 확인하세요

- AI가 명령어를 먼저 보여주면, 내용이 이해되지 않으면 실행하지 말고 먼저 질문하세요.
- `git status`로 어떤 파일이 올라가는지 항상 눈으로 확인하세요. [마무리 & 다음 단계](./wrap-up)에서 강조한 대외비 파일 주의사항이 여기서도 그대로 적용됩니다.
- `gh auth login` 단계에서는 브라우저 창이 열리고, 화면에 표시된 코드를 확인한 뒤 로그인하게 됩니다. 화면 안내를 따라가세요.

:::danger[대외비 파일 최종 점검]
결과물 폴더 안에 예산 자료, 개인정보, 비밀번호·API 키가 담긴 파일이 있다면 AI가 알려주는 목록을 꼭 확인하고, `.gitignore`에 등록되었는지 push 전에 다시 한번 확인하세요.
:::

## 완료 후 확인할 것

1. AI가 알려준 저장소 주소(예: `https://github.com/hyu-staff/hgd_mail-automation`)를 복사해두세요.
2. 브라우저에서 그 주소를 열어 파일들이 잘 올라갔는지 확인하세요.
3. 저장소가 **Private**으로 생성되었는지 확인하세요. (조직 구성원만 볼 수 있어야 합니다.)
